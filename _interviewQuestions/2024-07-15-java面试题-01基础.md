# 1. 对象创建过程

1. User user = new User();
2. 检查该条指令能否在常量池中定位到一个类的符号引用
3. 检查该符号引用代表的类是否被加载(类加载时, 可确定对象所需的内存)
4. 从java堆中划分对象所需的内存, 进行内存分配
5. 虚拟机初始化分配到的内存空间(保证实例字段在代码中未赋值便可使用)
6. 对对象进行设置(如该对象是哪个类的实例, 对象的哈希码等)



# 2. 对象的内存布局

1. 对象头
   1. MarkWord: 存储对象自身的运行时数据(GC分代年龄, 锁状态标识, 偏向线程id等)
   2. 类型指针: 对象具体是哪个类的实例, 会用指针指向对应的类元数据
   3. 如果对象是数组, 还会记录数组的长度
2. 实例数据
   1. 对象存储的有效信息, 代码中定义的各种类型的字段内容
3. 对齐填充
   1. 占位符, 对象实例数据部分如果没有对齐, 需要通过对齐填充补全



# 3. 对象的访问定位

对象的创建是为了使用,  Java程序需要通过栈上的reference数据来操作堆上的具体对象

访问对象的方式: 

1. 使用句柄: Java堆中会单独开辟一块空间, 句柄池, reference中存储的是对象的句柄地址, 句柄中包含了对象实例的地址和类型数据的地址。对象在移动时(垃圾回收)只用修改句柄中的实例数据指针, reference不用修改

2. 直接指针: reference中存储的是对象的地址。访问速度快, 不需要额外的指针定位。
